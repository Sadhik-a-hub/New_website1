2. Explain the Virtual DOM in React.

Answer: Virtual DOM is a lightweight copy of the Real DOM. React uses diffing and reconciliation to update only the changed parts of the DOM, improving performance. Example: User interacts with a React component. React updates the Virtual DOM instead of the actual DOM. It compares (diffs) the new Virtual DOM with the previous version. Only the changed elements get updated in the Real DOM.

5. What is Middleware in Express.js?

Answer: Middleware is a function that executes before reaching the final request handler in Express.js. Used for authentication, logging, error handling, and modifying requests. Example: const express = require('express'); const app = express(); // Middleware function app.use((req, res, next) => { console.log('Request received'); next(); // Pass control to next middleware }); app.get('/', (req, res) => { res.send('Hello, World!'); }); app.listen(3000);

6. Explain JWT Authentication in Node.js.

Answer: JWT (JSON Web Token) is used for secure user authentication. User logs in with credentials. Server creates a JWT token and sends it to the user. User sends JWT in Authorization header for subsequent requests. Server validates JWT before granting access. Example in Node.js: const jwt = require('jsonwebtoken'); const token = jwt.sign({ userId: 123 }, 'secretkey', { expiresIn: '1h' }); console.log(token);

8. How do you perform Pagination in MongoDB?

Answer: Pagination helps fetch limited results in large datasets. Example using skip() and limit(): db.users.find().skip(10).limit(5); skip(10) → Skips first 10 records. limit(5) → Fetches next 5 records.

9. What is Indexing in Databases?

Answer: Indexing improves the speed of data retrieval. Example in MySQL: CREATE INDEX idx_user ON users(name); Creates an index on the "name" column for faster searches.

15. How do you handle security in Full-Stack Applications?

Answer: 1. Authentication – Use JWT, OAuth, or session-based auth. 2. SQL Injection Prevention – Use parameterized queries. 3. XSS Prevention – Sanitize user inputs. 4. CORS Handling – Restrict cross-origin access. 5. HTTPS – Encrypt data using SSL/TLS.
